# Wheel Analytics — Android-застосунок

Нативний застосунок (Flutter), що показує ваші дані з сайту через JSON API (`api.php`).
Збирається у хмарі (GitHub Actions) — встановлювати Android Studio не потрібно.

## Що показує
- **Тиждень**: вибір тижня, премія по закритих опціонах, прогрес до цілі, графік факт/ціль, список угод (опціони/акції) з можливістю видалення
- **Календар**: премія по днях (відкриття) + реалізований P/L (закриття), перемикання місяців
- **Місяці**: премія по місяцях (графік + таблиця)
- Світла/темна тема

## Крок 0. Підготувати сайт (один раз)

1. Залийте на сайт файл **api.php** (у той самий корінь, де `index.php`).
2. У `config.php` додайте API-токен (будь-який довгий випадковий рядок):
   ```php
   'api_token' => 'тут_довгий_випадковий_рядок',
   ```
   Згенерувати можна так: `php -r "echo bin2hex(random_bytes(24));"` або просто вигадати ~30 символів.
3. Перевірте у браузері (підставте свій токен):
   `https://analitika.zakazat-kursovik.com/api.php?action=config&token=ВАШ_ТОКЕН`
   Має повернутися JSON виду `{"currency":"USD","target_pct":1.5,...}`. Якщо `unauthorized` — токен не збігається.

## Крок 1. Зібрати APK у хмарі (без встановлення інструментів)

1. Створіть безкоштовний акаунт на **github.com** (якщо немає).
2. Створіть новий **приватний** репозиторій (напр. `wheel-app`).
3. Завантажте в нього вміст цієї папки (`app_src/`, `.github/`) — через сайт GitHub: *Add file → Upload files* → перетягніть папки → Commit.
4. Перейдіть на вкладку **Actions** → workflow **Build APK** запуститься сам (або *Run workflow*).
5. Дочекайтеся зеленої галочки (~3–5 хв), відкрийте завершений запуск → блок **Artifacts** → завантажте **wheel-app-apk**. Усередині — `app-release.apk`.

## Крок 2. Встановити APK на телефон

1. Перекиньте `app-release.apk` на телефон (через USB, Telegram собі, Google Drive тощо).
2. Відкрийте файл на телефоні. Android попросить дозволити встановлення з невідомих джерел — дозвольте для свого файлового менеджера/браузера.
3. Встановіть. Зʼявиться іконка **Wheel Analytics**.

## Крок 3. Перший запуск

При першому відкритті застосунок попросить:
- **Адреса сайту** — `analitika.zakazat-kursovik.com` (можна без https://)
- **API-токен** — той, що ви вписали в `config.php`

Збережіть — і застосунок підтягне дані. Налаштування можна змінити будь-коли через ⚙ у шапці.

## Локальна збірка (альтернатива, якщо є Flutter)
Якщо колись поставите Flutter SDK:
```
flutter create --org com.wheel --project-name wheel_app build_app
cp app_src/pubspec.yaml build_app/pubspec.yaml
cp app_src/lib/main.dart build_app/lib/main.dart
cd build_app && flutter pub get && flutter build apk --release
```
APK буде в `build_app/build/app/outputs/flutter-apk/app-release.apk`.

## Безпека
- API-токен дає доступ до читання ваших торгових даних та видалення угод — тримайте застосунок і репозиторій приватними.
- Якщо токен «витік» — просто змініть рядок `api_token` у `config.php` і вкажіть новий у застосунку (⚙).
- Бажано, щоб сайт працював по HTTPS (cPanel/панель → SSL), тоді трафік застосунку шифрується.
